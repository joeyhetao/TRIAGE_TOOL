# 仿真日志分类分诊工具 — 产品需求文档（PRD）

**文档版本**：v1.1
**基准代码版本**：2026-03-08
**适用范围**：功能增改、需求评审、开发参考

---

## 1. 产品定位

面向芯片验证工程师的内网桌面工具，用于批量解析 UVM 仿真日志、自动匹配已知错误知识库、沉淀和复用排查经验，减少重复定位时间。

**部署形态**：单个 `triage_tool.exe`，双击运行，浏览器访问本地 Web 界面。知识库文件（`error_db.xlsx`）支持放置于网络共享盘供多人共用。

---

## 2. 用户与使用场景

| 角色 | 使用场景 |
|---|---|
| 验证工程师 | 上传仿真 log，查看自动分析结果，快速定位首错 |
| 资深工程师 | 将新错误根因写回知识库，积累团队经验 |
| 团队负责人 | 导出分析报告，归档或发送给相关人员 |

## 3. 当前功能（v1.0 基准）

### 3.1 日志上传与解析

- 支持拖拽或点击选择 `.log` / `.txt` 文件，可多选批量上传
- 解析每个日志文件，提取所有 `UVM_FATAL` / `UVM_ERROR` / `UVM_WARNING` 条目
- 每条错误记录包含：级别、时间戳、错误ID、文件位置、描述（含次行补充）
- 确定"首错"：取优先级最高（FATAL > ERROR > WARNING）的首次出现条目

**UVM 日志匹配正则**：
```
UVM_(ERROR|WARNING|FATAL) @ <时间戳> [<错误ID>] <文件位置>: <描述>
```

### 3.2 知识库匹配

两阶段匹配，对每个日志的首错执行：

1. **精确匹配**：错误ID（忽略大小写）+ 错误类型完全一致
2. **关键词匹配**：知识库条目的 `关键描述关键词`（逗号分隔）全部出现在首错描述中（AND 逻辑）

匹配结果状态：`matched`（命中）/ `unmatched`（未匹配）/ `no_error`（无报错）

### 3.3 结果展示

- 顶部汇总栏：日志总数、FATAL/ERROR/WARNING 总计、未匹配数
- 左侧文件导航列表，带彩色圆点（红/橙/黄）标示严重程度，未匹配显示徽章
- 右侧详情面板：错误统计、首错信息、知识库匹配结果

### 3.4 知识库回写

对未匹配日志，在结果页直接填写根因信息并写回知识库：

| 字段 | 必填 | 说明 |
|---|---|---|
| 关键描述关键词 | 否 | 逗号分隔，默认预填首错描述前50字符 |
| 报错原因 | **是** | 根因说明 |
| 所属模块 | 否 | 默认预填首错位置的首段 |
| 根因分类 | **是** | 枚举：DUT Bug / TB Bug / 用例问题 / 环境问题 |
| 解决方案 | 否 | 处理建议 |
| 关联用例 | 否 | 用例名称 |
| 录入人 | 否 | 姓名 |

写入时自动追加 `录入日期`（当天日期）。写操作通过双层锁（进程内线程锁 + 跨进程文件锁）保证并发安全。

### 3.5 报告导出

- **Excel 报告**：含"汇总"Sheet + 各日志独立 Sheet，带颜色样式
- **HTML 报告**：自包含单文件，可直接发送给无工具的人员查阅

### 3.6 知识库管理

- 默认知识库：`error_db.xlsx`（与 exe 同目录）
- 支持在 UI 填写自定义知识库路径（可指向网络共享盘）
- 知识库不存在时自动创建含样式表头的空白文件

## 4. 数据结构

### 4.1 知识库 Schema（`error_db.xlsx`）

| 列名 | 类型 | 说明 |
|---|---|---|
| 错误类型 | 字符串 | `UVM_FATAL` / `UVM_ERROR` / `UVM_WARNING` |
| 错误ID | 字符串 | UVM 日志中方括号内的 ID，用于精确匹配 |
| 关键描述关键词 | 字符串 | 逗号分隔，ALL 关键词均需命中（AND 逻辑） |
| 报错原因 | 字符串 | 根因说明 |
| 所属模块 | 字符串 | 出错模块名 |
| 根因分类 | 字符串 | DUT Bug / TB Bug / 用例问题 / 环境问题 |
| 解决方案 | 字符串 | 处理建议 |
| 关联用例 | 字符串 | 相关测试用例名 |
| 录入人 | 字符串 | 姓名 |
| 录入日期 | 字符串 | `YYYY-MM-DD`，写入时自动生成 |

### 4.2 内存数据结构（单次分析会话）

```python
# parse_logs() 返回，run_match() 追加 match 字段
{
  'file':       str,          # 原始文件名
  'filepath':   str,          # 服务器保存路径
  'statistics': {'UVM_FATAL': int, 'UVM_ERROR': int, 'UVM_WARNING': int},
  'first_error': {            # 可为 None
    'level':       str,       # UVM_FATAL / UVM_ERROR / UVM_WARNING
    'timestamp':   str,
    'error_id':    str,
    'location':    str,
    'description': str,
  },
  'all_errors': [... ],       # 同 first_error 结构，全部错误列表
  'match': {
    'status':   str,          # matched / unmatched / no_error
    'match_by': str,          # error_id / keywords / manual（仅 matched）
    'entry':    dict | None,  # 命中的知识库行
  }
}
```

## 5. 技术约束

| 约束项 | 说明 |
|---|---|
| 运行环境 | Windows，无网络访问 PyPI |
| 依赖 | `flask`、`openpyxl`（均通过离线 wheel 安装或打包进 exe） |
| 打包 | PyInstaller `--onefile`；`sys.frozen` 判断区分运行模式 |
| 并发写安全 | `threading.Lock`（进程内）+ `_FileLock`（跨进程，基于 `.lock` 文件 + `O_EXCL`） |
| 会话隔离 | 模块级 dict，key 为 Flask session 中的 UUID；上传文件名加 UUID 前缀 |
| 标准库优先 | 并发锁、报告 HTML 均使用标准库实现；新功能开发应优先评估标准库可行性 |

## 7. 已知限制与待决策项

| 编号 | 描述 | 当前状态 |
|---|---|---|
| L-01 | 会话数据存内存，重启 exe 后分析结果丢失，无法回溯历史 | 已知，设计如此 |
| L-02 | 仅取"首错"匹配，多条错误的全量匹配暂不支持 | 已知限制 |
| L-03 | 知识库只能追加，不支持在 UI 中编辑或删除已有条目 | 待评估 |
| L-04 | 上传文件永久保存在 `uploads/` 目录，无自动清理机制 | 待评估 |
| L-05 | `_FileLock` 超时（默认 15 秒）时写操作失败，前端提示错误但不重试 | 待评估 |
| L-06 | 根因分类为硬编码枚举，新增类别需改代码 | 待评估 |
